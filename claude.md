## Project Overview

Build a public, real-time dashboard that displays temperature and pressure data from a single ESP32 device. No authentication required — anyone with the link can access the dashboard.

### Key Requirements
- 1 ESP32 device posting real-time data every 5 seconds
- 5-10 concurrent users accessing the same dashboard simultaneously
- Public link (no login required)
- Real-time temperature & pressure display
- Historical data charts (24h, 7d, 30d views)
- CSV data export functionality
- Free hosting on Firebase

---

## Architecture

### Data Flow
```
ESP32 Device (every 5 sec)
    ↓
    POST JSON (temp, pressure, timestamp)
    ↓
Firebase Firestore (readings collection)
    ↓
React Dashboard (real-time listener)
    ↓
Public URL (anyone can access with link)
```

### Tech Stack
| Layer | Technology | Details |
|-------|-----------|---------|
| **Frontend** | React 18 + TypeScript | Existing Vite project + Shadcn UI |
| **Charts** | Recharts | Time-series visualization library |
| **Real-time Sync** | Firebase SDK | Firestore `.onSnapshot()` listeners |
| **Hosting** | Firebase Hosting | Free CDN + SSL included |
| **Backend** | Firestore | NoSQL document database |
| **Device Code** | C++ (Arduino) | ESP32 standard development |
| **Build Tool** | Vite | Already configured |
| **Data Export** | papaparse | CSV generation library |

## Firestore Schema

### Collection: `readings`

```
Document ID: auto-generated (Firebase creates)

Fields:
├── device_id: string ("esp32-lab-01")
├── temperature: number (float, e.g., 22.5)
├── pressure: number (float, e.g., 1013.25)
├── unit_temp: string ("Celsius" or "Fahrenheit")
├── unit_pressure: string ("hPa" or "inHg")
└── timestamp: Firestore Timestamp (Server-generated)
```

**Why this schema:**
- `device_id` for future device filtering
- `unit_*` fields store the unit so dashboard formats correctly
- `timestamp` auto-generated by Firestore (ensures consistency)
- No `devices` collection needed (single device only)

### Phase 3: ESP32 Arduino Code Development (1-2 hours)

**Required Libraries:**
- `WiFi.h` (built-in)
- `HTTPClient.h` (built-in)
- `ArduinoJson.h` (install via Arduino IDE Library Manager)
- Sensor library (DHT22, BME680, or whichever you're using)

**Core Functionality:**

```cpp
// Pseudocode structure
#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>
#include "DHT.h" // or your sensor

// Configuration
const char* ssid = "YOUR_SSID";
const char* password = "YOUR_PASSWORD";
const char* firestore_url = "https://firestore.googleapis.com/v1/projects/YOUR_PROJECT_ID/databases/(default)/documents/readings";
const char* api_key = "YOUR_API_KEY";
const char* device_id = "esp32-lab-01";
const int interval = 5000; // 5 seconds

DHT dht(GPIO_PIN, DHT22); // Adjust GPIO pin and sensor type

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  dht.begin();
  
  // Wait for WiFi connection
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("WiFi connected");
}

void loop() {
  if (WiFi.status() == WL_CONNECTED) {
    float temp = dht.readTemperature();
    float pressure = readPressure(); // Implement for your sensor
    
    postToFirestore(temp, pressure);
  }
  delay(interval);
}

void postToFirestore(float temp, float pressure) {
  HTTPClient http;
  String url = String(firestore_url) + "?key=" + api_key;
  
  http.begin(url);
  http.addHeader("Content-Type", "application/json");
  http.addHeader("X-Device-ID", device_id);
  
  // Build JSON payload
  DynamicJsonDocument doc(256);
  doc["fields"]["device_id"]["stringValue"] = device_id;
  doc["fields"]["temperature"]["doubleValue"] = temp;
  doc["fields"]["pressure"]["doubleValue"] = pressure;
  doc["fields"]["unit_temp"]["stringValue"] = "Celsius";
  doc["fields"]["unit_pressure"]["stringValue"] = "hPa";
  
  String payload;
  serializeJson(doc, payload);
  
  int httpCode = http.POST(payload);
  if (httpCode == 200) {
    Serial.println("Data posted successfully");
  } else {
    Serial.println("POST failed: " + httpCode);
  }
  
  http.end();
}
```

**Key Implementation Notes:**
- Use HTTPS (Firestore requires it)
- Include `X-Device-ID: esp32-lab-01` header (matches security rules)
- Post every 5 seconds (balances real-time feel with free tier limits)
- Add WiFi reconnection logic if connection drops
- Optional: Store failed posts in EEPROM and retry on reconnect

#### Cloud Function 1: Hourly Data Aggregation

**File:** `functions/src/aggregateReadings.ts`

**Trigger:** Scheduled (runs every hour via Cloud Scheduler)

**What it does:**
1. Queries all raw `readings` documents from the past hour
2. Computes min, max, and average for temperature and pressure
3. Writes a single summary document to a new `readings_hourly` collection
4. The dashboard queries `readings_hourly` for 7d and 30d views (instead of raw data)

**Firestore Schema — New Collection: `readings_hourly`**
```
Document ID: auto-generated

Fields:
├── device_id: string ("esp32-lab-01")
├── temp_avg: number (float)
├── temp_min: number (float)
├── temp_max: number (float)
├── pressure_avg: number (float)
├── pressure_min: number (float)
├── pressure_max: number (float)
├── reading_count: number (integer, how many raw readings were aggregated)
├── hour_start: Firestore Timestamp (start of the hour window)
└── hour_end: Firestore Timestamp (end of the hour window)
```

**Pseudocode:**
```typescript
import * as functions from 'firebase-functions';
import * as admin from 'firebase-admin';

admin.initializeApp();
const db = admin.firestore();

// Runs every hour at minute 5 (e.g., 1:05, 2:05, ...)
export const aggregateHourlyReadings = functions.pubsub
  .schedule('5 * * * *')
  .timeZone('UTC')
  .onRun(async () => {
    const now = new Date();
    const hourEnd = new Date(now.getFullYear(), now.getMonth(), now.getDate(), now.getHours(), 0, 0);
    const hourStart = new Date(hourEnd.getTime() - 60 * 60 * 1000);

    const snapshot = await db.collection('readings')
      .where('timestamp', '>=', admin.firestore.Timestamp.fromDate(hourStart))
      .where('timestamp', '<', admin.firestore.Timestamp.fromDate(hourEnd))
      .get();

    if (snapshot.empty) {
      console.log('No readings found for hour window');
      return null;
    }

    let tempSum = 0, tempMin = Infinity, tempMax = -Infinity;
    let pressSum = 0, pressMin = Infinity, pressMax = -Infinity;
    let count = 0;

    snapshot.forEach(doc => {
      const data = doc.data();
      tempSum += data.temperature;
      tempMin = Math.min(tempMin, data.temperature);
      tempMax = Math.max(tempMax, data.temperature);
      pressSum += data.pressure;
      pressMin = Math.min(pressMin, data.pressure);
      pressMax = Math.max(pressMax, data.pressure);
      count++;
    });

    await db.collection('readings_hourly').add({
      device_id: 'esp32-lab-01',
      temp_avg: tempSum / count,
      temp_min: tempMin,
      temp_max: tempMax,
      pressure_avg: pressSum / count,
      pressure_min: pressMin,
      pressure_max: pressMax,
      reading_count: count,
      hour_start: admin.firestore.Timestamp.fromDate(hourStart),
      hour_end: admin.firestore.Timestamp.fromDate(hourEnd),
    });

    console.log(`Aggregated ${count} readings for ${hourStart.toISOString()}`);
    return null;
  });
```

**Dashboard Changes (Phase 2 updates):**
- **24h view:** Query raw `readings` collection (up to ~17,280 docs -- still manageable for a single day)
- **7d view:** Query `readings_hourly` collection (~168 docs instead of ~120,960 raw docs)
- **30d view:** Query `readings_hourly` collection (~720 docs instead of ~518,400 raw docs)

This reduces Firestore reads for historical charts by **99.8%** on the 30d view.

#### Cloud Function 2: Scheduled Data Pruning

**File:** `functions/src/pruneReadings.ts`

**Trigger:** Scheduled (runs once daily at 2:00 AM UTC)

**What it does:**
1. Deletes all raw `readings` documents older than 48 hours (keeps 2 days of raw data for the 24h chart plus buffer)
2. Deletes all `readings_hourly` documents older than 31 days
3. Uses batched writes for efficient deletion (Firestore limit: 500 ops per batch)

**Pseudocode:**
```typescript
import * as functions from 'firebase-functions';
import * as admin from 'firebase-admin';

const db = admin.firestore();

// Runs daily at 2:00 AM UTC
export const pruneOldReadings = functions.pubsub
  .schedule('0 2 * * *')
  .timeZone('UTC')
  .onRun(async () => {
    const now = new Date();

    // Delete raw readings older than 48 hours
    const rawCutoff = new Date(now.getTime() - 48 * 60 * 60 * 1000);
    await deleteOldDocuments('readings', rawCutoff);

    // Delete hourly aggregates older than 31 days
    const hourlyCutoff = new Date(now.getTime() - 31 * 24 * 60 * 60 * 1000);
    await deleteOldDocuments('readings_hourly', hourlyCutoff, 'hour_start');

    return null;
  });

async function deleteOldDocuments(
  collectionName: string,
  cutoffDate: Date,
  timestampField: string = 'timestamp'
) {
  const cutoffTimestamp = admin.firestore.Timestamp.fromDate(cutoffDate);

  // Process in batches to avoid memory issues
  let totalDeleted = 0;
  let hasMore = true;

  while (hasMore) {
    const snapshot = await db.collection(collectionName)
      .where(timestampField, '<', cutoffTimestamp)
      .limit(500)
      .get();

    if (snapshot.empty) {
      hasMore = false;
      break;
    }

    const batch = db.batch();
    snapshot.docs.forEach(doc => batch.delete(doc.ref));
    await batch.commit();

    totalDeleted += snapshot.size;
    console.log(`Deleted batch of ${snapshot.size} from ${collectionName}`);

    // If we got fewer than 500, we're done
    if (snapshot.size < 500) {
      hasMore = false;
    }
  }

  console.log(`Pruned ${totalDeleted} documents from ${collectionName}`);
}
```

**Why 48 hours for raw data (not 24)?**
- The 24h chart needs a full 24 hours of raw data at any time
- The aggregation function runs hourly -- the extra 24-hour buffer ensures no raw data is deleted before it's been aggregated
- This guarantees the hourly rollup always has source data to work with

## Security Considerations

### Public Read Access
- Security rules allow anyone to read `readings` collection
- **This is intentional** — dashboard is meant to be public
- No sensitive user data is exposed

### ESP32 Write Protection
- Only device with header `X-Device-ID: esp32-lab-01` can write
- Always use HTTPS to prevent header spoofing
- API key is embedded in ESP32 code (acceptable for personal project)

### Future Enhancements (Optional)
- Add password protection to dashboard (add login page)
- Implement Firebase Auth for device authentication
- Restrict readings to specific time range or data size
